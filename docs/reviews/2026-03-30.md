## Bugs Fixed This Session

### 1. Extreme Value Entry Price Validation
**File:** `src/lib/strategies/extreme-value.ts`
**Bug:** Strategy checked `lastPrice` against threshold but computed `entryPrice` from `yes_bid/yes_ask`. On stale markets, these can diverge wildly — a 4c `lastPrice` qualifies, but `yes_bid` at 60c makes NO entry = 40c.
**Fix:** Added validation that `yes_bid/yes_ask` also support the extreme-value thesis before allowing entry.

### 2. Historical Trade Fetch Silent Errors
**File:** `src/app/api/trades/fetch-history/route.ts`
**Bug:** Catch block was `catch { errors++ }` — no error message captured. Also not using `normalizeTrade()` for Kalshi's `_dollars`/`_fp` response format.
**Fix:** Added error logging to catch block, imported and used `normalizeTrade()` for proper field normalization.

### 3. Mean Reversion Ticker Churn
**File:** `src/lib/strategies/engine.ts`
**Bug:** Same ticker (BOSMEM-225) was traded 4 separate times across scan cycles because the dedup only checked currently-open positions.
**Fix:** Added 48-hour per-ticker cooldown — won't re-enter any ticker that was traded in the last 48 hours.
